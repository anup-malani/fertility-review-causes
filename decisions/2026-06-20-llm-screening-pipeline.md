# Decision: Three-stage screening pipeline with iterative LLM calibration

**Date:** 2026-06-20
**Status:** Adopted

## Context

The 2-axis boolean query returns ~81,648 results for the old-age security hypothesis. Full corpus is not screenable by humans (would require ~820 person-hours at 1 min/abstract). A mechanism AND reduces the corpus but causes 83% false negative rate (see `2026-06-20-boolean-query-two-axis.md`). A different precision mechanism is needed.

## Decision

Adopt a three-stage pipeline:

**Stage 1 — Boolean search** (`literature-search.mjs`)
- Maximize recall; accept high false positive rate
- Output: raw corpus (potentially 50K–100K papers per hypothesis)

**Stage 2 — LLM calibration screen** (`calibrate-screen.mjs`, to implement)
- Iterative Haiku/Sonnet comparison on 1K-paper batches
- Haiku = primary screener (cheap, fast); Sonnet = gold standard/teacher
- Iterate until Haiku false negative rate vs. Sonnet < 3% for 2 consecutive batches
- Output: routing rule (which Haiku confidence levels to escalate to Sonnet), refined screening prompt, full corpus screened with `llm_verdict` field on each paper
- Cost: ~$10 calibration + ~$27 full run (Haiku + ~10% Sonnet escalation) per hypothesis

**Stage 3 — Human title/abstract screen** (`screen-titles-abstracts.mjs`)
- Human screens survivors from Stage 2
- Expected pass rate from Stage 2: ~15–25% of corpus (estimate)
- Human screening load: ~12,000–20,000 papers per hypothesis → split across 3 reviewers

## Cost asymmetry rationale

- False positives at Stage 1: cost = LLM tokens at Stage 2 (~$0.0002 per paper Haiku). Acceptable.
- False negatives at Stage 1: permanently lost papers. Unacceptable.
- False positives at Stage 2: cost = human screen time (~1 min per abstract). Manageable.
- False negatives at Stage 2: permanently lost papers. Unacceptable.
- Therefore: both Stage 1 and Stage 2 must optimize for recall, not precision. The "lean RELEVANT" tie-breaker instruction in the LLM screening prompt implements this at Stage 2.

## Screening prompt key design

The causal direction clause is the primary discriminating feature — it distinguishes papers where pensions affect fertility (relevant) from papers where fertility trends affect pension sustainability (not relevant). No keyword AND can capture this distinction. Validated on 24-paper test set: 100% recall, 100% precision.

Prompt stored at: `literature/search-logs/llm-screen-prompt.md`

## Reusability

The pipeline architecture reuses across all 61 hypotheses. What changes per hypothesis:
- The boolean query (Stages 1 output varies)
- The screening prompt hypothesis description and inclusion/exclusion criteria (auto-generated by `literature-search.mjs` dry-run)
- The routing rule (calibrated once per major category — see below)

**One calibration per major category:**
- Economic (C section): `old-age-security-pension-crowdout` is the pilot
- Biological (B section): one hypothesis TBD (e.g., `endocrine-disruptors-fecundity`)
- Cultural (D section): one hypothesis TBD (e.g., `secularization-religiosity`)
- Proximate (A section): one hypothesis TBD (e.g., `marriage-timing-union-formation`)
- Frameworks (E section): skip — use economic routing rule

The routing rule derived from each category calibration applies to all subsequent hypotheses in that category without a full re-calibration. The abstract structure of papers in a category is sufficiently homogeneous to generalize.

## Review date

Revisit if Stage 2 false negative rate (estimated from manual spot-checking of NOT_RELEVANT verdicts) exceeds 5% in production.
